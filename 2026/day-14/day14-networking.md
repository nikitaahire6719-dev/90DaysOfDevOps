# Day 14 – Networking Fundamentals & Hands-on Checks 🌐

Today I stepped into one of the most important foundations of DevOps—**Computer Networking**. Along with learning the core concepts, I performed hands-on networking diagnostics using common Linux troubleshooting commands.

## 📚 Concepts Covered

* OSI Model vs TCP/IP Model
* TCP vs UDP
* IP Addressing
* DNS, HTTP/HTTPS
* Network Interfaces (`lo`, `eth0`)
* ICMP and Ping
* Latency, Packet Loss & TTL

## 🛠️ Hands-on Commands Practiced

* `hostname -I`
* `ip addr show`
* `ping`
* `tracepath`
* `traceroute`
* `dig`
* `nslookup`
* `curl -I`
* `netstat`
* `ss`
* `nc`

## 🔍 Key Learnings

* Understood how devices communicate across a network.
* Learned the difference between the OSI and TCP/IP models.
* Used `ping` to verify connectivity and analyze latency.
* Traced packet routes using `tracepath` and `traceroute`.
* Performed DNS lookups with `dig` and `nslookup`.
* Verified HTTP responses using `curl`.
* Explored listening ports and active network connections.
* Verified service availability using `nc`.

## 💡 Interesting Observation

Several networking utilities were not installed by default on my Ubuntu system. Instead of stopping, I installed the required packages and continued the lab. It was a good reminder that troubleshooting and preparing the environment are also important DevOps skills.

Networking feels more theory-heavy than Linux at the moment, but I know this foundation will become valuable when I start working with Docker, Kubernetes, AWS, load balancers, and cloud networking.

Looking forward to learning more tomorrow! 🚀
